# RFC：基于 `chartType` 注释 + Markdown 表格的四象限图 {#rfc-quadrant-chart}

## 背景

业务上需要展示「四象限矩阵」式内容（如优先级矩阵：紧急/重要维度划分任务优先级、技术评估矩阵、竞品分析矩阵等），版式上常见为 **2×2 网格**，每个象限带标签并包含归属该象限的条目列表。

本仓库中，`docCards` 实现了「卡片栅格」，但 **未覆盖「按分类列将条目归入四象限」** 的版式需求。

本 RFC 约定：在**不破坏**现有注释与解析行为的前提下，用与 `docCards` **同一套作者写法** 支持「四象限图」。

## 目标

- **作者侧**：与 `docCards` 一致，在管道表上方写一行 JSON 注释，包含 `chartType: "quadrant"`。
- **数据侧**：**一行 Markdown 表 = 一个条目**；通过「象限」列的值将条目分组到四个区域。
- **呈现侧**：2×2 网格布局，每个区域带标签，包含该区域内的条目列表。

## 列契约

| 语义       | 建议表头                              | 是否必填 | 说明                                    |
| ---------- | ------------------------------------- | -------- | --------------------------------------- |
| 名称       | `名称`、`标题`、`项目`、`name`        | 是       | 条目名称                                |
| 象限       | `象限`、`分类`、`类别`、`quadrant`    | 是       | 所属象限；前 4 个不重复值构成四个区域    |
| 描述       | `描述`、`简介`、`说明`                | 否       | 条目描述，显示在名称下方                |

## 配置字段

| 字段        | 类型     | 说明                                               |
| ----------- | -------- | -------------------------------------------------- |
| `chartType` | `string` | 固定为 `"quadrant"`                                |
| `title`     | `string` | 图表标题（可选）                                   |
| `fieldMap`  | `object` | `{ name?, quadrant?, description? }` 字段映射（可选） |

## 注释示例

```markdown
<!-- {"chartType": "quadrant", "title": "优先级矩阵"} -->

| 名称           | 象限         | 描述           |
| :------------- | :----------- | :------------- |
| 修复线上bug    | 重要且紧急   | 影响用户       |
| 处理客户投诉   | 重要且紧急   | 优先处理       |
| 技术改进       | 重要不紧急   | 长期价值       |
| 回复邮件       | 不重要但紧急 | 日常事务       |
| 整理桌面       | 不重要不紧急 | 可以延后       |
```

## 与现有类型的关系

| 类型         | 作用                                       |
| ------------ | ------------------------------------------ |
| `docCards`   | 卡片栅格，不分组                           |
| `quadrant`   | **四象限矩阵**，按分类列将条目归入 2×2 网格 |
